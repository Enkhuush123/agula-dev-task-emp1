# REFLECTION

## 1) Энэхүү 3 хоногийн хугацаанд юу шинийг сурсан бэ?

Энэ даалгавраар би backend, frontend, data analysis-аа нэг repo дотор уялдуулж хийхийг илүү сайн ойлгосон.
Өмнө нь CRUD endpoint хийхдээ зөвхөн route/controller дээр төвлөрдөг байсан бол энэ удаа service, repository layer салгавал алдаа олоход амар байдаг гэдгийг мэдэрсэн.
Мөн premium calculation шиг бизнес дүрмийг code-д шууд бичихдээ validation-тэйгээ хамт бодох шаардлагатайг ойлгосон.
Frontend тал дээр API алдааг user-д ойлгомжтой хэлбэрээр харуулах, loading state үзүүлэх нь UX-д их нөлөөтэйг харлаа.
Day 3 дээр CSV анализ хийхдээ өгөгдөл дутуу үед assumption-оо ил тод бичих нь техникийн зөв хандлага гэдгийг сурсан.

## 2) Хамгийн хэцүү хэсэг нь юу байсан, яаж даван туулсан бэ?

Хамгийн хэцүү хэсэг нь business logic бичих байсан.
Ялангуяа premium тооцоолол, validation нөхцлүүдийг нэгтгэж зөв урсгалтай хийх нь надад шинэ зүйл байсан болохоор эхэндээ жаахан будилсан.
Тэгээд requirement-ээ дахин уншаад, хэсэг хэсгээр нь салгаж бичээд, тест маягаар endpoint-уудаа шалгаж явсаар шийдсэн.

## 3) Хэрэв дахин эхнээс нь хийх боломжтой бол юуг өөрөөр хийх байсан бэ?

Кодоо нэг дор их бичихээс илүү хэсэг хэсгээр нь хийж, бүр дээр нь шалгаад явах нь надад илүү зохих юм байна гэж ойлгосон.
Мөн README-гээ сүүлд биш өдөр бүр бага багаар бөглөж байх байсан, тэгвэл төгсгөлд их ачаалал үүсэхгүй.

## 4) Өгсөн хугацаандаа дуусгаж амжаагүй зүйлс, мөн тэдгээрийг цаашид хэрхэн сайжруулах санаа

Бүрэн automated test suite хийж амжаагүй.
Одоогоор manual шалгалтаар баталгаажуулсан ч production түвшинд бол API integration test заавал хэрэгтэй.
Мөн frontend дээр filter/sort/pagination нэмбэл бодит хэрэглээнд илүү ашигтай болно.
Security тал дээр input sanitation, rate limit, error logging стандартжуулах боломж байна.
Хэрвээ илүү цаг байсан бол Docker compose нэмээд нэг командаар database + app босдог болгох, бас deploy demo орчин гаргах байсан.
